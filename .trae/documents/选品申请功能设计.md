# 选品申请功能设计方案

## 📋 需求概述

1. **普通用户申请成为达人** → 自动通过（在user\_role表增加达人角色记录）
2. **达人申请选品商品进行邮寄** → 需要审核（审核通过后商家邮寄商品）
3. **选品申请表** → 即选品记录表（一张表同时记录申请和状态）

### 业务流程

#### 流程1：用户申请成为达人

```
1. 普通用户点击"申请成为达人"
   ↓
2. 系统自动通过
   ↓
3. user_role表增加一条达人角色记录
   ↓
4. 用户成为达人
```

#### 流程2：达人申请选品商品

```
1. 达人选择商品，点击"申请选品"
   ↓
2. 填写联系方式、手机号、邮寄地址
   ↓
3. 提交申请（状态：待审核）
   ↓
4. 商家/平台审核（通过/拒绝）
   ↓
5. 审核通过 → 商家邮寄商品
```

### 2. 选品申请表（核心表）

**表名：** `selection_apply`

**说明：** 这张表既是申请表，也是选品记录表，记录所有选品申请及状态。

```sql
-- ============================================
-- 选品申请表（也是选品记录表）
-- ============================================
CREATE TABLE `selection_apply` (
  `apply_id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '申请ID',
  
  -- 申请人信息
  `user_id` BIGINT NOT NULL COMMENT '申请人用户ID',
  `user_name` VARCHAR(50) COMMENT '申请人姓名',
  `contact_phone` VARCHAR(20) NOT NULL COMMENT '联系电话',
  `delivery_address` VARCHAR(500) NOT NULL COMMENT '邮寄地址',
  
  -- 商品信息
  `spu_id` BIGINT NOT NULL COMMENT '商品SPU ID',
  `spu_name` VARCHAR(200) COMMENT '商品名称（冗余）',
  `shop_id` BIGINT NOT NULL COMMENT '店铺ID',
  `shop_name` VARCHAR(100) COMMENT '店铺名称（冗余）',
  `main_img_url` VARCHAR(500) COMMENT '商品主图（冗余）',
  `price_fee` BIGINT COMMENT '商品价格（分）（冗余）',
  
  -- 申请信息
  `apply_remark` VARCHAR(500) COMMENT '申请备注/理由',
  `apply_time` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '申请时间',
  
  -- 审核信息
  `audit_status` TINYINT NOT NULL DEFAULT 1 COMMENT '审核状态 1-待审核 2-已通过 3-已拒绝',
  `audit_time` DATETIME COMMENT '审核时间',
  `audit_user_id` BIGINT COMMENT '审核人ID',
  `audit_remark` VARCHAR(500) COMMENT '审核备注',
  
  -- 物流信息
  `express_company` VARCHAR(50) COMMENT '快递公司',
  `express_no` VARCHAR(50) COMMENT '快递单号',
  `ship_time` DATETIME COMMENT '发货时间',
  
  -- 系统字段
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  `is_deleted` TINYINT(1) DEFAULT 0,
  
  PRIMARY KEY (`apply_id`),
  KEY `idx_user_id` (`user_id`),
  KEY `idx_spu_id` (`spu_id`),
  KEY `idx_shop_id` (`shop_id`),
  KEY `idx_audit_status` (`audit_status`),
  KEY `idx_apply_time` (`apply_time`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='选品申请表（选品记录表）';
```

#### 2. APP端 - 达人申请选品商品

**接口：** `POST /product/selection/apply`

**说明：** 达人申请选品商品进行邮寄，需要审核

**请求参数：**

```json
{
  "spuId": 123,                       // 商品ID（必填）
  "contactPhone": "13800138000",      // 联系电话（必填）
  "deliveryAddress": "北京市朝阳区xxx", // 邮寄地址（必填）
  "applyRemark": "希望试用这个产品"     // 申请备注（可选）
}
```

**响应：**

```json
{
  "code": 200,
  "msg": "申请成功，请等待审核",
  "data": {
    "applyId": 1
  }
}
```

**业务逻辑：**

1. 校验用户是否是达人角色
2. 检查是否重复申请（同一用户对同一商品）
3. 获取商品信息并冗余存储
4. 创建申请记录（audit\_status = 1 待审核）
5. 返回申请ID

## 📊 核心业务逻辑

### 1. 用户申请成为达人

```java
@Transactional
public void applyToBeTalent() {
    Long userId = AuthUserContext.get().getUserId();
    
    // 检查是否已是达人
    if (hasTalentRole(userId)) {
        throw new Mall4cloudException("您已经是达人了");
    }
    
    // 在user_role表增加达人角色记录
    UserRole userRole = new UserRole();
    userRole.setUserId(userId);
    userRole.setRoleId(100); // 达人角色ID
    userRoleMapper.insert(userRole);
}
```

### 2. 达人申请选品

```java
@Transactional
public Long applySelection(SelectionApplyDTO dto) {
    Long userId = AuthUserContext.get().getUserId();
    
    // 1. 校验用户是否是达人
    if (!hasTalentRole(userId)) {
        throw new Mall4cloudException("只有达人才能申请选品");
    }
    
    // 2. 检查是否重复申请
    if (hasApplied(userId, dto.getSpuId())) {
        throw new Mall4cloudException("您已申请过该商品");
    }
    
    // 3. 获取商品信息
    Spu spu = spuMapper.getBySpuId(dto.getSpuId());
    if (spu == null) {
        throw new Mall4cloudException("商品不存在");
    }
    
    // 4. 创建申请记录
    SelectionApply apply = new SelectionApply();
    apply.setUserId(userId);
    apply.setUserName(getUserName(userId));
    apply.setSpuId(dto.getSpuId());
    apply.setSpuName(spu.getName());
    apply.setShopId(spu.getShopId());
    apply.setMainImgUrl(spu.getMainImgUrl());
    apply.setPriceFee(spu.getPriceFee());
    apply.setContactPhone(dto.getContactPhone());
    apply.setDeliveryAddress(dto.getDeliveryAddress());
    apply.setApplyRemark(dto.getApplyRemark());
    apply.setApplyTime(new Date());
    apply.setAuditStatus(1); // 待审核
    selectionApplyMapper.insert(apply);
    
    return apply.getApplyId();
}
```

### 3. 审核选品申请

```java
@Transactional
public void auditSelection(Long applyId, SelectionAuditDTO dto) {
    SelectionApply apply = selectionApplyMapper.getById(applyId);
    if (apply == null) {
        throw new Mall4cloudException("申请记录不存在");
    }
    
    if (apply.getAuditStatus() != 1) {
        throw new Mall4cloudException("该申请已审核，无法重复审核");
    }
    
    Long auditorId = AuthUserContext.get().getUserId();
    
    apply.setAuditStatus(dto.getAuditStatus());
    apply.setAuditTime(new Date());
    apply.setAuditUserId(auditorId);
    apply.setAuditRemark(dto.getAuditRemark());
    
    selectionApplyMapper.update(apply);
}
```

## ⚠️ 注意事项

1. **权限校验**

   * 申请选品：必须是达人角色

   * 审核选品：商家或平台管理员

   * 填写物流：商家或平台管理员

2. **幂等性**

   * 同一用户对同一商品只能有一条待审核或已通过的申请

   * 拒绝后可以重新申请

3. **数据冗余**

   * 商品信息冗余存储，避免关联查询

   * 用户信息冗余存储

4. **达人角色**

   * role表增加 `talent` 角色

   * user\_role表关联用户和达人角色
