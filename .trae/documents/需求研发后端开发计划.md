# 需求研发后端开发计划

## 一、需求概述

为前端已创建的需求管理页面（demand-index、demand-submit、demand-detail）开发完整的后端支持。

### 前端页面已就绪
| 页面 | 路径 | 功能 |
|------|------|------|
| demand-index | /pages/demand-index/demand-index | 需求首页、Banner、快捷入口、最近需求列表 |
| demand-submit | /pages/demand-submit/demand-submit | 研发需求表单提交 |
| demand-detail | /pages/demand-detail/demand-detail | 需求详情、撤回、重新申请、删除 |

### 开发范围
- **后端**：`mall4cloud-platform` 模块（复用现有模块，不创建新模块）
- **商户管理端页面**：`mall4cloud-multishop-refactor/apps/example`

## 二、API 设计

### APP端接口（mall4cloud-user 或独立服务）

| 接口 | 方法 | 路径 | 功能 |
|------|------|------|------|
| 需求列表 | GET | /demand/list | 获取需求列表 |
| 需求详情 | GET | /demand/detail/{id} | 获取需求详情 |
| 提交需求 | POST | /demand/submit | 提交新需求 |
| 撤回需求 | POST | /demand/withdraw | 撤回需求 |
| 重新申请 | POST | /demand/reapply | 重新申请 |
| 删除需求 | POST | /demand/delete | 删除需求 |

### 商户管理端接口（mall4cloud-platform）

| 接口 | 方法 | 路径 | 功能 |
|------|------|------|------|
| 需求管理列表 | GET | /demand/page | 分页查询需求 |
| 需求处理 | POST | /demand/handle | 处理需求（确认/研发/采样/完成） |
| 需求详情 | GET | /demand/{id} | 获取需求详情（管理端） |

## 三、数据库表

执行 `rd_demand.sql`：
```sql
CREATE TABLE `rd_demand` (
  `id` INT NOT NULL AUTO_INCREMENT,
  `demand_no` VARCHAR(50) NOT NULL COMMENT '需求编号',
  `title` VARCHAR(200) NOT NULL COMMENT '需求标题',
  `functional_appeal` TEXT NOT NULL COMMENT '功能诉求',
  `target_audience` VARCHAR(200) NOT NULL COMMENT '目标人群',
  `dosage_form_preference` VARCHAR(100) DEFAULT NULL COMMENT '剂型偏好',
  `budget_range` VARCHAR(50) NOT NULL COMMENT '预算范围',
  `expected_delivery_time` DATE NOT NULL COMMENT '期望交付时间',
  `remark` TEXT DEFAULT NULL COMMENT '备注',
  `submitter_id` VARCHAR(50) NOT NULL COMMENT '提交人ID',
  `submitter_name` VARCHAR(50) DEFAULT NULL COMMENT '提交人姓名',
  `submitter_phone` VARCHAR(20) DEFAULT NULL COMMENT '提交人电话',
  `status` TINYINT NOT NULL DEFAULT 0 COMMENT '状态: 0-待处理 1-确认中 2-研发中 3-样品制作 4-已完成 5-已取消',
  `status_text` VARCHAR(20) NOT NULL COMMENT '状态文本',
  `admin_remark` TEXT DEFAULT NULL COMMENT '处理备注',
  `handler_name` VARCHAR(50) DEFAULT NULL COMMENT '处理人',
  `submit_time` DATETIME NOT NULL COMMENT '提交时间',
  `update_time` DATETIME NOT NULL COMMENT '更新时间',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_demand_no` (`demand_no`),
  KEY `idx_submitter_id` (`submitter_id`),
  KEY `idx_status` (`status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='研发需求主表';
```

## 七、状态流转

```
0-待处理 → 1-确认中 → 2-研发中 → 3-样品制作 → 4-已完成
    ↓           ↓
  5-已取消   5-已取消
```

| 状态值 | 状态文本 | 可执行操作 |
|--------|----------|-----------|
| 0 | 待处理 | 确认、处理 |
| 1 | 确认中 | 转为研发中 |
| 2 | 研发中 | 转为样品制作 |
| 3 | 样品制作 | 完成 |
| 4 | 已完成 | - |
| 5 | 已取消 | 重新申请 |

## 八、注意事项

1. **分布式ID**：使用 `SegmentFeignClient` 获取需求编号
2. **用户关联**：APP端通过 `submitterId` 关联用户
3. **管理端权限**：需登录商户账号
4. **Nacos配置**：在 Nacos 中确认数据库连接配置
