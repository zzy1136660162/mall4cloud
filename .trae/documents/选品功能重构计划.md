# 选品功能重构计划

> **For agentic workers:** REQUIRED SUB-SKILL: 使用 superpowers:subagent-driven-development (recommended) 或 superpowers:executing-plans 来实施此计划。

**目标：** 将选品相关功能（选品列表、详情、申请样品、申请列表、申请详情）从源项目迁移到 mall4cloud-uniapp 主项目

**架构：** 采用页面级重构，适配目标项目的 Vue3 + uni-app 语法风格，保持 UI/UX 一致性

**技术栈：** uni-app (Vue3 Composition API)、SCSS

***

## 二、页面目录结构（3级）

采用统一的 3 级目录结构，所有选品相关页面统一放在 `pages/sample/` 目录下：

```
pages/
└── sample/                              # 样品/选品主目录
    ├── selection/                        # 选品子模块
    │   ├── selection.vue                # 选品列表（TabBar 入口）
    │   └── selection.scss
    ├── product/                          # 商品详情（选品场景）
    │   ├── product.vue
    │   └── product.scss
    ├── apply/                            # 申请样品
    │   ├── apply.vue
    │   └── apply.scss
    ├── list/                             # 我的样品列表
    │   ├── list.vue
    │   └── list.scss
    └── detail/                           # 样品申请详情
        ├── detail.vue
        └── detail.scss
```

### 命名规范说明

* **一级目录** `sample/` - 样品/选品业务主目录

* **二级目录** - 功能模块划分

  * `selection/` - 选品（浏览商品）

  * `product/` - 商品详情

  * `apply/` - 申请样品

  * `list/` - 我的样品列表

  * `detail/` - 样品申请详情

* **文件命名** - 与二级目录名保持一致，统一使用 kebab-case

## 六、技术要点

### 1. Vue3 语法适配

* `data()` → `ref()` / `reactive()`

* `methods` → `function` in `setup()`

* `this.setData()` → 直接赋值

* `v-for` 中的 `index` 参数调整

### 2. 样式兼容

* 保持 rpx 单位一致性

* 复用目标项目的颜色变量（如有）

* 保持圆角、间距规范

* 每个页面使用独立的 `.scss` 文件

### 3. 组件复用

* 地区选择器需新建或适配

* 暂无新增公共组件需求
