# 需求管理页面迁移计划

## 一、迁移概述

将旧版小程序页面 `demandDetail`、`demandSubmit`、`rd_index` 迁移到新的 uni-app 商城框架 `mall4cloud-uniapp`。

### 旧版页面功能分析

| 页面 | 功能 | API端点 |
|------|------|---------|
| rd_index | 首页入口、Banner、快捷入口、服务流程、最近需求列表 | GET /demand/list |
| demandSubmit | 研发需求表单提交 | POST /demand/submit |
| demandDetail | 需求详情查看、撤回、重新申请、删除 | GET /demand/detail/{id}<br>POST /demand/withdraw<br>POST /demand/reapply<br>POST /demand/delete |

### 新框架技术栈

- Vue 3 Composition API (`<script setup>`)
- uni-app (H5/小程序多端)
- `http.js` 封装请求，使用 `uni.getStorageSync('cloudToken')` 获取token
- SCSS 样式

---

## 二、迁移步骤

### 步骤1：页面配置 - 更新 pages.json

在 `src/pages.json` 中添加3个新页面路径：
- `pages/demand-index/demand-index` (原rd_index)
- `pages/demand-submit/demand-submit`
- `pages/demand-detail/demand-detail`

### 步骤2：创建需求首页 - demand-index.vue

创建 `src/pages/demand-index/demand-index.vue`：

**功能：**
- Banner区域（从旧版迁移）
- 快捷入口：提交需求、我的需求（后续可扩展人才库）
- 服务流程展示
- 最近需求列表（调用API获取前3条）

**API调用：**
- `GET /demand/list` - 获取需求列表

**交互：**
- 点击"提交需求" → 跳转 `/pages/demand-submit/demand-submit`
- 点击"我的需求" → 跳转 `/pages/demand-list/demand-list`（需后续创建）
- 点击最近需求项 → 跳转详情页

### 步骤3：创建需求提交页 - demand-submit.vue

创建 `src/pages/demand-submit/demand-submit.vue`：

**功能：**
- 研发需求表单
- 必填字段：需求标题、功能诉求、目标人群、预算范围、期望交付时间
- 选填字段：剂型偏好、备注、联系人姓名、联系电话
- 表单验证
- 提交成功后跳转列表页

**API调用：**
- `POST /demand/submit` - 提交需求

### 步骤4：创建需求详情页 - demand-detail.vue

创建 `src/pages/demand-detail/demand-detail.vue`：

**功能：**
- 状态卡片（待处理/确认中/研发中/采样中/已完成/已取消）
- 需求信息展示
- 处理信息展示（处理人、处理备注）
- 提交信息展示
- 操作按钮：根据状态显示"查看进度"、"撤回需求"、"重新申请"、"删除"

**API调用：**
- `GET /demand/detail/{id}?submitterId=` - 获取需求详情
- `POST /demand/withdraw` - 撤回需求
- `POST /demand/reapply` - 重新申请
- `POST /demand/delete` - 删除需求

### 步骤5：在"我的"页面添加入口

更新 `src/pages/my/my.vue`，在"我的工具"区域添加"我的需求"入口。

### 步骤6：后端API适配

新框架的http请求规范：
- 使用 `http.request({url, method, data})`
- 自动携带 `Authorization: cloudToken`  header
- 响应处理：`responseData.code === '00000'` 表示成功，数据在 `responseData.data`
- 错误码：A00001/A00002/A00004等需提示用户

**API路径前缀需确认：** 根据新后端微服务架构，demand相关API可能需要添加服务前缀（如 `/mall4cloud_demand/`），具体需与后端确认。

## 四、注意事项

1. **API后缀确认**：旧版API `/demand/*` 在新框架中是否需要添加前缀，需后端确认
2. **Token获取**：使用 `uni.getStorageSync('cloudToken')` 替代旧版 `app.globalData.userId`
3. **状态值映射**：旧版状态值 0-5 对应不同状态，需保持一致
4. **页面跳转**：部分跳转页面（如demandList、talentPool）如有需要也应一并迁移
5. **图片资源**：复用的图片如"我的需求.png"需迁移到新框架的static目录
