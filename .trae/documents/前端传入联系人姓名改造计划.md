# 将联系人姓名改为从前端传入

## 问题描述
当前 `SelectionApplyServiceImpl.applySelection()` 使用：
```java
String userName = AuthUserContext.get().getNickName();
```
从后端上下文获取用户昵称，但 `UserInfoInTokenBO` 中没有 `nickName` 字段，需要改为从前端小程序传入联系人姓名。

## 修改方案

### 1. 修改 SelectionApplyDTO.java
**文件位置**: `mall4cloud-platform/.../dto/SelectionApplyDTO.java`

**添加字段**:
```java
@NotBlank(message = "联系人姓名不能为空")
@Schema(description = "联系人姓名")
private String userName;
```

### 2. 修改 SelectionApplyServiceImpl.java
**文件位置**: `mall4cloud-platform/.../service/impl/SelectionApplyServiceImpl.java`

**修改内容**:
```java
// 原代码：
Long userId = AuthUserContext.get().getUserId();
String userName = AuthUserContext.get().getNickName();

// 修改为：
Long userId = AuthUserContext.get().getUserId();
String userName = dto.getUserName(); // 从前端传入
```

## 预期交付物
1. `SelectionApplyDTO.java` - 添加 `userName` 字段及 getter/setter
2. `SelectionApplyServiceImpl.java` - 修改获取用户姓名的逻辑

## 前端小程序修改提示
前端小程序在调用选品申请接口时，需要在请求体中添加 `userName` 字段（联系人姓名）。
