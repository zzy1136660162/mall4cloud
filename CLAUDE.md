## Project Overview

mall4cloud is a microservice-based B2B2C e-commerce platform built with Spring Cloud 2025.x and Vue 3, licensed under AGPLv3.

## Development Environment

### Prerequisites
- **Java**: 17
- **Node.js**: >=16 (pnpm >7 for frontend)
- **Middleware**: MySQL 8.0, Redis 7.0, Nacos 3.1.1, Seata 2.6.0, Elasticsearch 7.17.21, RocketMQ 5.2.0, MinIO, Canal

### Quick Start
1. Start middleware: `docker-compose -f doc/中间件docker-compse一键安装/docker-compose.yaml up -d`
2. Build backend: `mvn clean package -DskipTests` from project root
3. Start services via scripts (see below)
4. Start frontend: `pnpm install && pnpm dev` in each front-end directory

### Service Scripts
- `scripts/start.sh` — start all or a single service (logs to `scripts/logs/`)
- `scripts/stop.sh` — graceful stop (SIGTERM then SIGKILL)
- `scripts/restart.sh` — serial or `--parallel` restart
- `scripts/services.conf` — central config: JAR name, port, JVM flags per service

**Startup order** (from `scripts/services.conf`):
LEAF → USER → RBAC → AUTH → PRODUCT → ORDER → PAYMENT → SEARCH → BIZ → MULTISHOP → PLATFORM → GATEWAY

## Architecture

### Microservices Structure
```
mall4cloud-api/          # Feign interface definitions (internal calls)
mall4cloud-common/       # Shared modules: cache, core, database, order, rocketmq, security
mall4cloud-gateway/      # API gateway, routing, rate limiting
mall4cloud-auth/         # Authentication, token management
mall4cloud-{service}/    # Business services
```

### Module Naming
- **mall4cloud-api-{module}**: Feign client interfaces for service-to-service calls
- **mall4cloud-common-{type}**: Shared utilities and configs

### Code Layers (per service)
```
controller/     # REST endpoints (admin/, app/, platform/ subdirs)
  ├─ admin/     # Merchant backend APIs (/admin/...)
  ├─ app/       # Mobile app APIs (/m/...)
  └─ platform/  # Platform backend APIs (/platform/...)
service/        # Business logic interfaces
  └─ impl/      # Implementations
manager/        # Cross-cutting business logic
mapper/         # MyBatis data access
model/          # Database entities
dto/            # Data transfer objects
vo/             # View objects
```

## Key Patterns

### API Response
Use `ServerResponseEntity<T>` from `com.mall4j.cloud.common.response`:
- Success: `ServerResponseEntity.success(data)` or `ServerResponseEntity.success()`
- Fail: `ServerResponseEntity.fail(ResponseEnum.XXX)` or `ServerResponseEntity.showFailMsg(msg)`

Response codes: `00000` = success, `A00004` = unauthorized

### Authentication
- Token passed via `Authorization` header
- `AuthUserContext.get()` retrieves current user info in any context
- Feign internal calls require `/feign` prefix and secret header
- Use `SeataRequestInterceptor` for distributed transaction XID propagation

### Distributed ID
Use `@DistributedId` annotation on ID fields. IDs generated by mall4cloud-leaf service (Snowflake).

### Redis Caching
Cache names defined in `mall4cloud-common-cache/.../constant/`. Use `CacheNames` constants.

### Database
- MyBatis 4.0.1 with annotation-based mappers
- `PageDTO` / `PageVO` for pagination
- `BaseModel` for entity inheritance (createTime, updateTime, etc.)
- SQL schemas in `db/` directory per service

### Frontend API Calls
```javascript
import request from '@/utils/request'
// Returns res.data directly, throws on error
const data = await page(params)
```

Frontend env vars: `VITE_APP_BASE_API` (gateway URL), `VITE_APP_RESOURCES_URL` (file storage), `VITE_APP_RESOURCES_TYPE` (0=Aliyun, 1=MinIO)

## Adding New Features

### New Feign Client
1. Define interface in `mall4cloud-api-{module}/src/main/java/.../feign/`
2. Annotate with `@FeignClient(contextId = "...", value = "mall4cloud-{service}")`
3. Add `@GetMapping`/`@PostMapping` etc. with path
4. In consuming service, add dependency on api module

### New MyBatis Mapper
1. Create interface in `mapper/` with `@Mapper` annotation
2. Create XML in `resources/mapper/` with namespace matching interface
3. Use `@Param` for multiple parameters

### New Controller Endpoint
1. Place in appropriate subdirectory (admin/app/platform)
2. Return `ServerResponseEntity<T>`
3. Use `@Operation` annotation for Swagger docs
4. Use `@Valid` for request body validation
