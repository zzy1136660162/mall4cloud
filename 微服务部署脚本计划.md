## 脚本设计

### 文件结构
```
mall4cloud/
├── scripts/                    # 脚本目录
│   ├── start.sh               # 启动脚本
│   ├── stop.sh                # 停止脚本
│   ├── restart.sh             # 重启脚本
│   ├── status.sh              # 状态检查脚本
│   └── services.conf          # 服务配置文件
├── apps/                      # jar包目录（用户需要将编译好的jar放入）
└── logs/                      # 日志目录
```

### 脚本功能要求

#### 1. start.sh - 启动脚本
**功能：**
- 启动所有微服务（默认）
- 支持启动单个指定服务：`./start.sh mall4cloud-auth`
- 后台运行模式
- 每个服务独立的日志文件
- PID文件管理
- 启动顺序控制（gateway最后启动）
- 启动前检查端口是否被占用

**使用示例：**
```bash
./start.sh              # 启动所有服务
./start.sh mall4cloud-gateway  # 只启动网关
```

#### 2. stop.sh - 停止脚本
**功能：**
- 优雅停止所有服务（默认）
- 支持停止单个指定服务：`./stop.sh mall4cloud-auth`
- 使用kill -15优雅停止
- 等待超时后使用kill -9强制停止
- 清理PID文件
- 按启动顺序的反序停止服务

**使用示例：**
```bash
./stop.sh              # 停止所有服务
./stop.sh mall4cloud-gateway  # 只停止网关
```

#### 3. restart.sh - 重启脚本
**功能：**
- 重启所有服务（默认）
- 支持重启单个服务：`./restart.sh mall4cloud-auth`

**使用示例：**
```bash
./restart.sh           # 重启所有服务
./restart.sh mall4cloud-auth  # 只重启认证服务
```

#### 4. status.sh - 状态检查脚本
**功能：**
- 检查所有服务状态（默认）
- 检查单个服务状态：`./status.sh mall4cloud-auth`
- 显示：服务名、PID、端口、运行状态、启动时间、内存使用

**使用示例：**
```bash
./status.sh            # 查看所有服务状态
./status.sh mall4cloud-auth  # 查看认证服务状态
```

#### 5. services.conf - 服务配置文件
**内容：**
```bash
# 服务名称:JAR包名:启动端口:JVM参数
GATEWAY=mall4cloud-gateway:8000:-Xms256m -Xmx512m
AUTH=mall4cloud-auth:9101:-Xms256m -Xmx512m
...
```

## 脚本实现细节

### 通用功能模块
1. **日志管理**
   - 每个服务独立的日志文件：`logs/{服务名}.log`
   - 支持日志轮转（通过logback配置）
   - 彩色输出状态信息

2. **PID管理**
   - PID文件位置：`pids/{服务名}.pid`
   - 进程检查和验证

3. **错误处理**
   - 启动失败检测
   - 端口冲突检测
   - 重复启动检测
   - 超时控制

4. **JVM参数配置**
   - 默认JVM参数：`-Xms256m -Xmx512m`
   - 可在配置文件中自定义
   - GC日志配置

### 启动顺序
```
第1批：依赖最少的服务
1. mall4cloud-leaf (基础ID生成服务)
2. mall4cloud-user (用户服务)
3. mall4cloud-rbac (权限服务)

第2批：业务服务
4. mall4cloud-auth (认证服务)
5. mall4cloud-product (商品服务)
6. mall4cloud-order (订单服务)
7. mall4cloud-payment (支付服务)
8. mall4cloud-search (搜索服务)
9. mall4cloud-biz (业务服务)
10. mall4cloud-multishop (多店铺服务)
11. mall4cloud-platform (平台服务)

第3批：网关
12. mall4cloud-gateway (网关服务 - 最后启动)
```

### 脚本内容

#### start.sh 核心逻辑
```bash
# 1. 创建必要的目录
# 2. 加载服务配置
# 3. 检查参数（单个服务 or 所有服务）
# 4. 按顺序启动服务
# 5. 每个服务启动时：
#    - 检查端口是否被占用
#    - 检查PID文件是否存在
#    - 后台启动 java -jar 命令
#    - 等待服务启动
#    - 验证服务是否成功启动
# 6. 显示启动结果
```

#### stop.sh 核心逻辑
```bash
# 1. 加载服务配置
# 2. 检查参数（单个服务 or 所有服务）
# 3. 按反向顺序停止服务
# 4. 每个服务停止时：
#    - 读取PID文件
#    - 发送 SIGTERM 信号
#    - 等待超时（默认30秒）
#    - 发送 SIGKILL 信号（如果超时）
#    - 删除PID文件
# 5. 显示停止结果
```

## 用户使用指南

### 快速开始
1. 编译项目生成jar文件
2. 将jar文件复制到 `apps/` 目录
3. 运行启动脚本：`bash start.sh`

### 环境要求
- Linux/Unix系统
- Bash 4.0+
- JDK 8 或更高版本
- 已安装Maven（用于编译）

### 配置文件
编辑 `services.conf` 可以：
- 修改JVM内存参数
- 调整启动超时时间
- 启用/禁用特定服务

### 监控和维护
```bash
# 查看所有服务状态
./status.sh

# 查看特定服务日志
tail -f logs/mall4cloud-auth.log

# 重启单个服务
./restart.sh mall4cloud-auth
```
