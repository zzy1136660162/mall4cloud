# 杰东认证官网首页设计规格

## 背景

在当前空目录中创建一个服务端渲染的 Vue 前端项目，用于开发“杰东认证”官网。品牌主线按“杰东认证”处理，不按“杰东优选”电商或产品平台处理。

参考信息：

- 原站：http://www.bjjdjy.com.cn/
- 视觉参考：https://www.yuanxinjituan.com/
- 用户确认的视觉方向：B，医疗健康集团式明亮官网
- 用户确认的技术要求：服务端渲染、TailwindCSS、纯前端展示、mock 数据

## 目标

先交付一个可运行的首页，体现高端、专业、可信的医药健康认证服务官网形象。首页要能作为后续扩展“关于我们、服务领域、政策资讯、案例、联系我们”等页面的基础。

## 非目标

- 不接真实后端接口。
- 不迁移旧站 ASP 程序。
- 不开发后台管理系统。
- 不开发完整多页面官网。
- 不做在线表单真实提交，只展示联系入口和咨询动作。

## 技术选型

使用 `Nuxt 3 + Vue 3 + TypeScript + TailwindCSS`。

Nuxt 3 负责 Vue 生态下的服务端渲染、路由、SEO 元信息和构建流程。TailwindCSS 负责首页视觉系统、响应式布局和组件样式。首页数据先集中放在本地 mock 数据文件中，后续接 CMS 或接口时只替换数据来源。

## 视觉方向

采用明亮医疗健康集团式官网风格，参考元心集团官网的健康行业集团感，但内容和信息层级围绕杰东认证重构。

视觉原则：

- 主色使用医疗青绿，辅以深海蓝文字和浅青灰背景。
- 页面保持大面积留白，避免老站式密集表格布局。
- 卡片圆角控制在 8px 左右，视觉保持克制、专业。
- 使用轻微渐变、数据卡、服务矩阵和流程模块建立现代感。
- 首页第一屏必须明确传达“医药健康产品批文批号申办服务伙伴”。
- 不使用夸张营销页、紫色渐变、装饰性大色块或无关插画。

## 首页信息架构

### 顶部导航

内容：

- Logo：杰东认证
- 导航：首页、关于杰东、服务领域、申办流程、政策资讯、成功案例、联系我们
- 右侧按钮：立即咨询

移动端导航折叠为菜单按钮。

### Hero 首屏

主标题：

医药健康产品批文批号申办服务伙伴

副文案：

聚焦保健食品、化妆品、消毒产品、医疗器械等注册备案与合规咨询，提供资料审核、申报路径设计和全过程节点跟进服务。

操作按钮：

- 咨询服务
- 查看申办流程

首屏右侧展示一组专业化业务卡片，例如“保健食品注册”“消毒产品备案”“化妆品备案”“医疗器械咨询”。

### 服务领域

使用 mock 数据展示服务卡片：

- 保健食品注册与备案
- 化妆品备案与申报
- 消毒产品卫生安全评价
- 医疗器械注册备案
- 国药准字与药品相关咨询
- 食字号、QS、特种批文咨询

每张卡片包含标题、简短说明、关键词标签。

### 为什么选择杰东

展示四项能力：

- 申报路径设计
- 资料审核与清单管理
- 政策法规跟踪
- 全流程节点跟进

该模块用于增强专业可信度，不使用夸大承诺。

### 申办流程

流程节点：

1. 需求评估
2. 资料准备
3. 申报递交
4. 过程跟进
5. 结果交付

桌面端展示为横向流程，移动端展示为纵向时间线。

### 政策资讯

使用 mock 新闻列表，方向参考原站已有栏目：

- 行业资讯
- 政策法规
- 药品知识
- 食品与保健食品
- 化妆品
- 医疗器械
- 消毒产品

首页只展示 4 到 6 条资讯，包含分类、标题、日期和摘要。

### 案例或成果展示

展示 mock 案例数据，强调服务类型和项目状态，不展示敏感客户信息。

示例：

- 保健食品注册资料整理项目
- 国产非特殊化妆品备案咨询
- 消毒产品卫生安全评价资料辅导
- 一类医疗器械备案路径咨询

### 页脚

包含：

- 北京杰东认证服务有限公司
- 业务范围简述
- mock 联系方式：电话、邮箱、地址先使用可替换的示例内容
- 快速导航
- ICP 备案信息：开发阶段使用“京ICP备12045069号-1”作为 mock 文案，正式上线前由用户确认真实备案信息

## 数据设计

首页 mock 数据放在 `data/home.ts`，导出结构化对象：

- `hero`
- `services`
- `advantages`
- `processSteps`
- `news`
- `cases`
- `footer`

组件只消费 mock 数据，不在组件内硬编码列表内容。

## 组件边界

首页拆分为聚焦组件：

- `AppHeader.vue`：顶部导航和移动端菜单
- `HeroSection.vue`：首屏内容和业务卡片
- `ServiceGrid.vue`：服务领域卡片
- `AdvantageSection.vue`：选择杰东的理由
- `ProcessTimeline.vue`：申办流程
- `NewsSection.vue`：政策资讯列表
- `CaseShowcase.vue`：案例或成果展示
- `AppFooter.vue`：页脚

`pages/index.vue` 负责组合组件和传入数据。

## SEO 与 SSR

首页使用 Nuxt 的 `useSeoMeta` 设置：

- title：杰东认证 - 医药健康产品批文批号申办服务伙伴
- description：北京杰东认证服务有限公司提供保健食品、化妆品、消毒产品、医疗器械等注册备案与合规咨询服务。
- keywords：杰东认证, 北京杰东, 批文批号, 保健食品注册, 化妆品备案, 消毒产品, 医疗器械备案

构建产物应支持 SSR 运行，开发阶段使用 Nuxt dev server 验证。

## 响应式要求

- 桌面端首页首屏左右分栏。
- 平板端降低卡片密度。
- 手机端导航折叠，首屏改为单列，服务卡片和资讯列表纵向展示。
- 页面元素不得重叠，按钮文字不得溢出。

## 验收标准

- 项目可通过 `npm run dev` 启动。
- 首页在桌面和移动宽度下布局正常。
- 页面使用 Nuxt 3 SSR 框架。
- 页面使用 TailwindCSS 构建主要样式。
- 首页数据来自 mock 数据文件。
- 首页包含导航、Hero、服务领域、优势、流程、资讯、案例、页脚。
- 页面视觉符合已确认的 B 方向：明亮、医疗健康集团式、专业可信。

## 风险与处理

- 参考站为前端渲染站点，静态抓取内容有限；方案只借鉴视觉与结构气质，不复制其实现。
- 原站为旧 ASP/table 布局；新项目不迁移旧代码，只提炼业务内容和栏目方向。
- 当前目录初始不是 Git 仓库；如果需要按 Superpowers 流程提交文档和后续代码，需要先初始化 Git 仓库或由用户提供现有仓库。
