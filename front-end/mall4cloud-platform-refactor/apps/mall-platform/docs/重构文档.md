### 2.1 登录模块

| 旧项目 | 新项目 | 迁移方式 |
|--------|--------|----------|
| `src/views/common/login/index.vue` | `src/views/login.vue` | **替换文件** |
| `src/stores/modules/user.js` | `src/store/modules/app/account.ts` | **合并逻辑** |
| `src/api/auth/auth.js` | `src/api/modules/app.ts` | **合并接口** |
| `src/utils/auth.js` | `src/utils/storage.ts` | **替换文件** |
| `src/permission.js` | `src/router/guards.ts` | **框架已实现** |

**迁移步骤：**

1. 替换 `src/views/login.vue` 的内容，参考旧项目登录页的 UI 交互逻辑（用户名/密码/验证码），但复用新项目的样式
2. 修改 `src/store/modules/app/account.ts`：
   - `login()` 调用旧项目的 `/mall4cloud_auth/ua/login` 接口
   - `logout()` 调用旧项目的 `/mall4cloud_auth/login_out` 接口
   - 用户信息结构对齐旧项目的 `user.js`
3. 在 `src/api/modules/app.ts` 中添加登录相关接口

**旧项目 auth API：**
```javascript
// src/api/auth/auth.js
import request from '@/utils/request'
export const login = (data) => request.post('/mall4cloud_auth/ua/login', data)
export const loginOut = () => request.post('/mall4cloud_auth/login_out')
export const getUserInfo = () => request.get('/mall4cloud_auth/user info') // 根据实际接口调整
```

**新项目 account store 需调整：**
```typescript
// src/store/modules/app/account.ts
async login(loginData: object) {
  const { data } = await apiApp.login(loginData) // 调用合并后的登录接口
  this.token = data.token
  this.setToken(data.token)
  await this.getUserInfo()
  await this.getPermissions()
  return data
}
```

## 五、关键注意事项

### 1. 布局框架差异
旧项目使用自定义 `layout/` 目录，新项目使用 Fantastic-admin 的 `layouts/` 框架。页面组件不需要引用 layout，layout 由路由中的 `Layout` 组件自动包裹。

### 2. 样式兼容性
旧项目页面组件可能依赖旧的 layout 样式变量。迁移后以新框架为主，部分 `.el-*` 样式可能需要调整。

### 3. Store 架构差异
- 旧项目：`src/stores/modules/user.js` + `src/stores/modules/permission.js`
- 新项目：`src/store/modules/app/account.ts` + `src/store/modules/app/route.ts`
- **建议**：以新项目 store 为基础，将旧项目 store 中的业务逻辑合并进去

### 4. 动态路由加载时机
旧项目在 `permission.js` 中路由守卫加载，新项目在 `src/store/modules/app/route.ts` 的 `generateRoutes()` 中。确保菜单 API 返回的路由格式与新项目的路由定义格式兼容。

### 5. 文件上传
旧项目 `src/api/biz/oss.js` 的上传接口需要在新项目中保持一致。上传组件直接复制即可。

### 6. Token 存储
旧项目用 cookie 存储 token，新项目用 localStorage。确认后端 token 格式不变，只需调整存储方式（`src/utils/storage.ts` 已封装）。
