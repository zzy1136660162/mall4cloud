# Docker Compose 生产环境配置优化

## 变更说明

本次优化将所有中间件服务的IP地址从服务器公网IP(`115.191.79.224`)改为本地环回地址(`127.0.0.1`)，以提升性能和稳定性。

## 修改的文件清单

### 1. docker-compose.yaml

**修改内容**：
- `MINIO_BROWSER_REDIRECT_URL`: `115.191.79.224` → `127.0.0.1`
- `MYSQL_SERVICE_HOST` (Nacos): `115.191.79.224` → `127.0.0.1`
- RocketMQ Broker 启动命令: `-n 115.191.79.224:9876` → `-n 127.0.0.1:9876`
- RocketMQ Dashboard: `JAVA_OPTS` 中的namesrv地址
- Seata `SEATA_IP`: `115.191.79.224` → `127.0.0.1`

**影响**：所有Docker服务使用host网络模式，内部通信改为本地访问。

### 2. seata/application.yml

**修改内容**：
- 数据库连接URL: `115.191.79.224:3306` → `127.0.0.1:3306`

**影响**：Seata服务连接MySQL使用本地地址。

### 3. rocketmq/broker/conf/broker.conf

**修改内容**：
- `namesrvAddr`: `115.191.79.224:9876` → `127.0.0.1:9876`
- `brokerIP1`: `115.191.79.224` → `127.0.0.1`

**影响**：RocketMQ Broker使用本地地址注册。

### 4. canal/conf/canal.properties

**修改内容**：
- `rocketmq.namesrv.addr`: `115.191.79.224:9876` → `127.0.0.1:9876`

**影响**：Canal连接RocketMQ使用本地地址。

### 5. canal/conf/example/instance.properties

**修改内容**：
- `canal.instance.master.address`: `115.191.79.224:3306` → `127.0.0.1:3306`

**影响**：Canal连接MySQL使用本地地址。

## 网络架构变化

### 优化前（使用公网IP）

```
应用服务 ──→ 115.191.79.224:3306 (MySQL) ──→ 公网
应用服务 ──→ 115.191.79.224:6379 (Redis) ──→ 公网
应用服务 ──→ 115.191.79.224:9876 (RocketMQ) ──→ 公网
```

### 优化后（使用本地环回）

```
应用服务 ──→ 127.0.0.1:3306 (MySQL) ──→ 本地
应用服务 ──→ 127.0.0.1:6379 (Redis) ──→ 本地
应用服务 ──→ 127.0.0.1:9876 (RocketMQ) ──→ 本地
```

## 技术原理

### Docker Host网络模式

所有中间件服务使用`network_mode: "host"`，这意味着：

- 容器直接使用宿主机网络栈
- 容器端口直接映射到宿主机端口
- 容器间通信通过`127.0.0.1`（localhost）
- 不需要Docker内部网络

### 本地环回地址优势

- **零网络开销**: 数据包不经过网卡
- **零延迟**: 直接在内核空间转发
- **高可靠**: 不受网络波动影响
- **安全**: 外部无法直接访问

## 注意事项

1. **仅限单服务器部署**: 此优化适用于所有服务部署在同一台服务器
2. **集群场景**: 如果中间件分布在多台服务器，需要相应调整
3. **防火墙配置**: 确保服务器防火墙允许必要端口
