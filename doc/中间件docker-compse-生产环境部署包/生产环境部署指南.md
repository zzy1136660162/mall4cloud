# Mall4Cloud 生产环境部署指南

## 概述

本指南帮助您在生产环境中部署Mall4Cloud电商系统的中间件服务。所有配置已优化为使用**本地环回地址(127.0.0.1)**，以最大化性能和稳定性。

## 架构优势

### 为什么使用本地环回地址？

使用`127.0.0.1`（本地环回地址）而非服务器公网IP有以下优势：

1. **性能提升**：本地通信绕过网络栈，减少延迟
2. **稳定性增强**：不受公网网络波动影响
3. **安全性提高**：减少外部网络攻击面
4. **配置简化**：所有容器使用host网络模式，内部通信无阻隔

## 服务组件

### 包含的中间件服务

| 服务 | 端口 | 说明 | 配置文件 |
|------|------|------|----------|
| MySQL | 3306 | 主数据库 | mysql/conf.d/docker.cnf |
| Redis | 6379 | 缓存服务 | - |
| Nacos | 8848 | 注册中心/配置中心 | docker-compose.yaml |
| MinIO | 9000/9001 | 对象存储 | docker-compose.yaml |
| RocketMQ | 9876/10911 | 消息队列 | rocketmq/broker/conf/broker.conf |
| Seata | 8091 | 分布式事务 | seata/application.yml |
| Canal | 11111 | 数据同步 | canal/conf/*.properties |
| Nginx | 80/443 | 反向代理 | nginx/conf/nginx.conf |

## 部署步骤

### 1. 环境准备

确保服务器已安装：

- Docker 20.10+
- Docker Compose 2.0+
- 至少4GB可用内存
- 50GB可用磁盘空间

```bash
# 检查Docker版本
docker --version
docker compose version

# 检查系统资源
df -h
free -h
```

### 2. 上传配置文件

将整个`中间件docker-compse-生产环境部署包`目录上传到服务器。

**重要**：所有配置文件已预设为`127.0.0.1`，无需手动替换IP地址。

```bash
# 创建部署目录
mkdir -p /opt/mall4cloud
cd /opt/mall4cloud

# 上传并解压配置文件
# (使用scp、rsync或其他方式上传)
```

### 3. 权限设置

```bash
# 进入部署目录
cd /opt/mall4cloud/中间件docker-compse-生产环境部署包

# 设置目录权限
chmod -R 777 ./rocketmq
chmod -R 666 ./minio/data
chmod -R 777 ./elasticsearch/data 2>/dev/null || true
```

### 4. 启动服务

```bash
# 拉取镜像并启动所有服务（后台运行）
docker compose up -d --build

# 查看服务状态
docker compose ps

# 查看日志（排除Nginx日志）
docker compose logs --tail=100 | grep -v nginx
```

**预计耗时**：首次部署需要下载约7GB镜像，请耐心等待。

### 5. 验证服务状态

```bash
# 检查所有容器是否正常运行
docker compose ps

# 查看容器健康状态
docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"
```

## MinIO存储桶配置

### 1. 登录MinIO管理界面

访问 `http://127.0.0.1:9001`，使用管理员账号登录。

### 2. 创建存储桶

1. 点击左侧"Buckets"
2. 点击"Create Bucket"
3. 输入桶名称：`mall4cloud`
4. 点击"Create Bucket"确认

### 3. 设置桶权限

1. 选择刚创建的`mall4cloud`桶
2. 进入"Access Rules"
3. 添加规则：
   - Prefix: `*`
   - Access: `public`

### 4. 设置访问策略

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": {"AWS": ["*"]},
      "Action": ["s3:GetObject"],
      "Resource": ["arn:aws:s3:::mall4cloud/*"]
    }
  ]
}
```

## Elasticsearch索引配置

### 创建必要索引

使用Postman或curl创建ES索引：

**地址**: `http://127.0.0.1:9200`

**认证**: 无需认证（本地环境）

需要创建以下索引：

1. `product` - 商品索引
2. `order` - 订单索引

详细mapping配置请参考项目中的：
- `es/product.md`
- `es/order.md`

### 创建索引命令

```bash
# 检查ES是否正常
curl http://127.0.0.1:9200

# 创建product索引
curl -X PUT "http://127.0.0.1:9200/product" \
  -H "Content-Type: application/json" \
  -d @product_mapping.json

# 创建order索引
curl -X PUT "http://127.0.0.1:9200/order" \
  -H "Content-Type: application/json" \
  -d @order_mapping.json
```

## Nacos配置

### 初始化配置

配置文件位于 `mysql/initdb/` 目录下的SQL文件已包含：

1. `mall4cloud_nacos.sql` - Nacos配置数据
2. `mall4cloud_canal.sql` - Canal配置数据
3. `mall4cloud_seata.sql` - Seata配置数据

这些SQL文件会在MySQL首次启动时自动执行。

### 修改默认密码

首次登录Nacos后，请立即修改默认密码：

1. 访问 `http://127.0.0.1:8848/nacos`
2. 使用 `nacos/nacos` 登录
3. 进入"权限控制" → "用户管理"
4. 修改nacos用户的密码

## 性能优化建议

### JVM参数调整

根据服务器内存调整JVM配置：

```yaml
# Nacos JVM配置（docker-compose.yaml）
environment:
  - JVM_XMS=512m    # 建议设为可用内存的1/4
  - JVM_XMX=512m

# RocketMQ JVM配置
environment:
  JAVA_OPT_EXT: "-Duser.home=/home/rocketmq -Xms1g -Xmx1g -Xmn512m"
```

### MySQL优化

编辑 `mysql/conf.d/docker.cnf`：

```ini
[mysqld]
# 根据可用内存调整
innodb_buffer_pool_size = 1G
max_connections = 500
query_cache_size = 64M
```

### Redis优化

```bash
# 设置最大内存
docker exec mall4cloud-redis redis-cli -a Gesoft9919. CONFIG SET maxmemory 2gb

# 启用AOF持久化
docker exec mall4cloud-redis redis-cli -a Gesoft9919. CONFIG SET appendonly yes
```
